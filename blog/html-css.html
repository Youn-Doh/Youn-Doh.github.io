What are the best practices associated with using classes vs. ids?

'class' and 'id' are two different attributes used in HTML and they are selectors used in CSS. They both belong to 'elements' (tags), and they work similar where we can make our work simple.

So different between those two are

  • 'id' is unique selector
  • 'class' is a selector you can use muptilple times in different tags
For example, when you make an id for signing up a website or something, it has to be unique. On the other hand when you think of a class, it is a some sort of groups.
Ask yourself "is there more than one element which requires the same style, now or at any time in the future?", and the answer is even "maybe", then make it a class.

Examples

How to use them in HTML and CSS?

'id'

<html>
       
       <p id="hello-world">Hellow World</p>
       
       <style>
       
       #hello-world {
         color: white;
         background-color: red;
       }
After opening 'p' element and you can type 'id' with quote marks. Then in CSS style sheet, use '#' to set "Hellow World" as white text color and red background.

'class'

<html>
     
     <p class="red-background">Hello World</p>
     <div class="red-background">Hello Universe</div>
     
     <style>
     
     .red-background {
       background-color: red;
     }
It is same until you type 'class' with the quote marks, then when you go to the style sheet this time use'.' in front of class to set background as red.
